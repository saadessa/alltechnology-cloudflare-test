# أفضل إعدادات Continue للمبرمجين: دليل التحسين 2026

**Continue.dev** هو مساعد برمجة بالذكاء الاصطناعي مفتوح المصدر وقوي، لكن للحصول على أفضل تجربة، تحتاج إلى تهيئته بشكل صحيح. الإعدادات الصحيحة يمكن أن تزيد من إنتاجيتك بشكل كبير، وتحسن جودة الاقتراحات، وتسرع الاستجابة.

هذا الدليل يقدم لك أفضل إعدادات Continue.dev لتحسين تجربة البرمجة بالذكاء الاصطناعي في 2026.

---

## 1. ملف الإعدادات (config.json)

ملف الإعدادات الرئيسي لـ Continue.dev هو `config.json`. يمكنك الوصول إليه من خلال:

1. فتح Continue في VS Code (أيقونة الشريط الجانبي).
2. الضغط على أيقونة **الإعدادات** (الترس).
3. اختيار **"Open Config"**.

---

## 2. إعدادات النماذج (Models)

### 2.1 اختيار النماذج المناسبة

| الاستخدام | النموذج الموصى به | السبب |
|-----------|-------------------|-------|
| **الإكمال التلقائي** | CodeLlama 7B أو DeepSeek-Coder 6.7B | خفيف وسريع، اقتراحات جيدة |
| **المحادثات** | GPT-4 أو Claude أو DeepSeek-Coder 33B | فهم عميق، ردود دقيقة |
| **التشغيل المحلي** | CodeLlama 7B أو Mistral 7B | مجاني، خاص، سريع |

### 2.2 إعداد نموذجين (خفيف وثقيل)

```json
{
  "models": [
    {
      "title": "CodeLlama 7B (Autocomplete)",
      "provider": "ollama",
      "model": "codellama:7b"
    },
    {
      "title": "DeepSeek-Coder 33B (Chat)",
      "provider": "ollama",
      "model": "deepseek-coder:33b"
    }
  ],
  "tabAutocompleteModel": {
    "title": "CodeLlama 7B",
    "provider": "ollama",
    "model": "codellama:7b"
  }
}
```

### 2.3 إعداد النماذج السحابية

```json
{
  "models": [
    {
      "title": "GPT-4",
      "provider": "openai",
      "model": "gpt-4",
      "apiKey": "YOUR_API_KEY"
    },
    {
      "title": "Claude 3.5 Sonnet",
      "provider": "anthropic",
      "model": "claude-3-5-sonnet-20241022",
      "apiKey": "YOUR_ANTHROPIC_API_KEY"
    }
  ]
}
```

## 3. إعدادات الإكمال التلقائي (Autocomplete)

### 3.1 تمكين الإكمال التلقائي المتعدد الأسطر

```json
{
  "tabAutocompleteOptions": {
    "multilineCompletions": "always"
  }
}
```

**الخيارات**:
- `"always"`: دائماً (موصى به).
- `"never"`: أبداً.
- `"auto"`: تلقائي.

### 3.2 ضبط سرعة الإكمال التلقائي

```json
{
  "tabAutocompleteOptions": {
    "debounceDelay": 200,
    "maxPromptTokens": 1024
  }
}
```

- **debounceDelay**: تأخير قبل عرض الاقتراح (200ms موصى به).
- **maxPromptTokens**: الحد الأقصى لرموز السياق للإكمال التلقائي (1024-2048).

### 3.3 تخصيص لغة الإكمال التلقائي

يمكنك تحديد لغات برمجة معينة للإكمال التلقائي:

```json
{
  "tabAutocompleteOptions": {
    "language": {
      "python": true,
      "javascript": true,
      "typescript": true,
      "go": true,
      "rust": true
    }
  }
}
```

## 4. إعدادات السياق (Context)

### 4.1 تفعيل سياق الكود الكامل

```json
{
  "contextProviders": [
    {
      "name": "codebase",
      "params": {
        "maxTokens": 4096
      }
    }
  ]
}
```

### 4.2 تفعيل سياق المحطة الطرفية

```json
{
  "contextProviders": [
    {
      "name": "terminal",
      "params": {}
    }
  ]
}
```

### 4.3 تفعيل سياق المستندات

```json
{
  "contextProviders": [
    {
      "name": "docs",
      "params": {
        "paths": ["README.md", "docs/"]
      }
    }
  ]
}
```

### 4.4 ضبط حجم السياق

```json
{
  "contextLength": 4096
}
```

**التوصيات**:
- **أجهزة محدودة**: 2048
- **أجهزة متوسطة**: 4096
- **أجهزة قوية**: 8192+

### 4.5 تمكين السياق التلقائي

```json
{
  "automaticContext": true
}
```

## 5. اختصارات لوحة المفاتيح (Keybindings)

### 5.1 الاختصارات الافتراضية

| الاختصار (Windows/Linux) | الاختصار (macOS) | الوظيفة |
|--------------------------|------------------|---------|
| `Ctrl+Shift+L` | `Cmd+Shift+L` | فتح نافذة المحادثة |
| `Ctrl+Shift+I` | `Cmd+Shift+I` | فتح محادثة مضمّنة |
| `Tab` | `Tab` | قبول الاقتراح |
| `Ctrl+Space` | `Cmd+Space` | عرض اقتراحات إضافية |

### 5.2 تخصيص الاختصارات

لتخصيص الاختصارات:

1. في VS Code، اضغط `Ctrl+K Ctrl+S`.
2. ابحث عن **"Continue"**.
3. عدّل الاختصارات حسب رغبتك.

**اختصارات موصى بها**:

```json
[
  {
    "key": "ctrl+shift+c",
    "command": "continue.openChat"
  },
  {
    "key": "ctrl+shift+i",
    "command": "continue.inlineChat"
  },
  {
    "key": "ctrl+shift+a",
    "command": "continue.acceptSuggestion"
  }
]
```

## 9. نصائح لتحسين الأداء

### 9.1 استخدم النماذج المحلية

النماذج المحلية عبر Ollama أسرع وأكثر خصوصية من النماذج السحابية.

> **للمزيد**: راجع [Continue مع Ollama: التشغيل المحلي](/posts/continue-ollama).

### 9.2 اختر النموذج المناسب لجهازك

| حجم الجهاز | النموذج الموصى به |
|------------|-------------------|
| **محدود (8GB RAM)** | Phi-3 Mini أو CodeLlama 7B |
| **متوسط (16GB RAM)** | CodeLlama 13B أو Mistral 7B |
| **قوي (32GB+ RAM)** | DeepSeek-Coder 33B أو CodeLlama 34B |

### 9.3 قلل حجم السياق للأجهزة المحدودة

```json
{
  "contextLength": 2048
}
```

### 9.4 استخدم نموذجين مختلفين

- **نموذج خفيف**: للإكمال التلقائي.
- **نموذج ثقيل**: للمحادثات.

### 9.5 فعّل تسريع GPU

إذا كان لديك بطاقة رسوميات NVIDIA، تأكد من تثبيت CUDA وتشغيل Ollama مع دعم GPU.

### 9.6 أغلق التطبيقات غير الضرورية

حرر الذاكرة والمعالج لتسريع النماذج المحلية.

## 10. حل المشكلات الشائعة في الإعدادات

### 10.1 الإكمال التلقائي لا يعمل

**الحل**:
1. تأكد من تعيين `tabAutocompleteModel`.
2. تحقق من اتصال النموذج (Ollama أو API).
3. تأكد من تفعيل الإكمال التلقائي في الإعدادات.

### 10.2 النموذج بطيء جداً

**الحل**:
1. استخدم نموذجاً أصغر.
2. قلل `contextLength`.
3. فعّل تسريع GPU.

### 10.3 الأخطاء في ملف config.json

**الحل**:
1. تحقق من صيغة JSON باستخدام مدقق JSON.
2. تأكد من صحة أسماء النماذج.
3. أعد تشغيل VS Code بعد التعديل.

> **للمزيد**: راجع [دليل Continue.dev الشامل](/posts/continue-dev-guide) و [حل مشكلات ChatGPT الشائعة](/posts/chatgpt-common-problems).

## 11. الخلاصة

إعدادات Continue.dev الصحيحة يمكن أن تحدث فرقاً كبيراً في تجربة البرمجة بالذكاء الاصطناعي. من خلال اختيار النماذج المناسبة، ضبط الإكمال التلقائي، تخصيص السياق، وتحسين الأداء، يمكنك تحقيق أقصى استفادة من هذه الأداة القوية.

**الخلاصة في نقاط**:
- **النماذج**: استخدم نموذجاً خفيفاً للإكمال التلقائي وثقيلاً للمحادثات.
- **الإكمال التلقائي**: فعّل الإكمال المتعدد الأسطر وضبط سرعة الظهور.
- **السياق**: فعّل سياق الكود الكامل والمحطة والمستندات.
- **الاختصارات**: خصص اختصارات مريحة للوصول السريع.
- **الأداء**: استخدم النماذج المحلية وقلل حجم السياق للأجهزة المحدودة.
- **التخصيص**: استفد من الأوامر المخصصة لتحسين سير العمل.

**توصيتنا**: ابدأ بملف الإعدادات المتكامل أعلاه وعدّله حسب احتياجاتك وجهازك. جرّب نماذج مختلفة واختر الأنسب لك.
