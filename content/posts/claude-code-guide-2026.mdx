# Claude Code: الدليل الشامل للمبتدئين والمحترفين في 2026

في عام 2026، لم تعد أدوات البرمجة بالذكاء الاصطناعي مجرد مكملات للكود. أصبحت وكلاء (Agents) مستقلين يمكنهم التخطيط والتنفيذ عبر قاعدة الكود بالكامل.

**Claude Code** من Anthropic هي واحدة من أقوى هذه الأدوات للمبرمجين. صممت لتكون رفيقك في الطرفية (Terminal) الذي يقرأ مشروعك، يخطط للتغييرات، وينفذها عبر ملفات متعددة، ويرسل طلبات السحب (Pull Requests) - كل ذلك في جلسة واحدة.

هذا الدليل يغطي كل ما تحتاج معرفته عن Claude Code: شرح Claude Code، التثبيت، الميزات، التكامل مع VS Code، خطط الأسعار، وحالات الاستخدام العملية للمبرمجين.

---

## ما هو Claude Code؟

Claude Code هو نظام برمجة وكيل (Agentic Coding System) من Anthropic. الفكرة الأساسية: **الطرفية هي الواجهة، قاعدة الكود هي السياق، والوكيل (Agent) يدير سير العمل من البداية إلى النهاية**.

**باختصار:** تصف ما تريد بناءه أو إصلاحه بلغة طبيعية، ويقوم Claude Code Agent بتنفيذ المهمة عبر ملفات متعددة، مع الحفاظ على الشفافية والتحكم الكامل.

للمبتدئين الذين يرغبون في البدء خطوة بخطوة، يمكنك الاطلاع على [شرح Claude Code للمبتدئين: دليل كامل لخطواتك الأولى](/posts/claude-code-beginners-guide).

### المبادئ الأساسية لاستخدام Claude Code

- **وكيل وليس مكمل:** Claude Code لا يقترح السطر التالي فقط. إنه يقرأ قاعدة الكود بأكملها، يخطط لنهج عبر ملفات متعددة، ينفذ التغييرات، يدير الاختبارات، ويكرر حتى تنجح المهمة.
- **الطرفية أولاً:** يعيش Claude Code في الطرفية ويتكامل مع أدوات Shell الموجودة. Claude Code CLI هي التجربة الأساسية، بينما إضافات VS Code و JetBrains تجلبه إلى المحرر.
- **سياق قاعدة الكود بالكامل:** بدلاً من طلب تحديد الملفات يدوياً، يستخدم Claude Code البحث الوكيل لفهم هيكل المشروع وتبعياته تلقائياً.
- **شفاف وآمن:** يطلب Claude Code الموافقة قبل تغيير الملفات أو تشغيل الأوامر. يمكنك الموافقة على كل تغيير على حدة، أو تفعيل الوضع التلقائي (Auto Mode) للمهام الموثوقة.

## الميزات الرئيسية لـ Claude Code

### 1. وكلاء متوازون (Parallel Sub-agents)

يمكن لـ Claude Code تشغيل وكلاء فرعيين متوازيين لأداء مهام متعددة في وقت واحد. هذا يسمح بمعالجة مهام ضخمة مثل البحث عن الأخطاء في قاعدة كود كاملة أو إجراء تحسينات أداء موزعة.

**التحديث الأخير (يونيو 2026):** يمكن للوكلاء الفرعيين الآن إنشاء وكلاء فرعيين خاصين بهم (حتى 5 مستويات عمق).

### 2. سير العمل الديناميكي (Dynamic Workflows)

الميزة الأكثر ثورية في Claude Code. يمكنه كتابة نصوص تنسيق (Orchestration Scripts) تدير عشرات إلى مئات من الوكلاء الفرعيين في جلسة واحدة.

**مثال واقعي:** استخدم فريق Jarred Sumner سير العمل الديناميكي لترجمة Bun من Zig إلى Rust. أنتجوا 750,000 سطر من كود Rust مع اجتياز 99.8% من مجموعة الاختبارات الحالية، في 11 يوماً فقط.

### 3. عرض الوكيل (Agent View)

أمر `claude agents` يفتح شاشة واحدة تعرض كل جلسة Claude Code، وتظهر ما يعمل، وما هو محظور عليك، وما تم إنجازه.

### 4. وضع التلقائي (Auto Mode)

يحل وضع التلقائي محل طلبات الموافقة بفحوصات أمان خلفية. متاح الآن على حسابات Pro ويدعم Sonnet 4.6 و Opus. كما يتوفر على Bedrock و Vertex و Foundry.

### 5. نقاط الاستعادة (Checkpoints)

يمكنك التراجع عن الأخطاء بسهولة. تعمل نقاط الاستعادة مثل "التراجع" (Undo) للمشاريع بأكملها.

### 6. CLAUDE.md: السياق الثابت

ملف `CLAUDE.md` هو ملف تكوين يقرأه Claude Code في بداية كل جلسة. يضاف إلى التعليمات النظامية قبل رسالتك الأولى، مما يمنح Claude معرفة دائمة بمشروعك دون الحاجة لإعادة شرح أي شيء.

### 7. إدارة الاختبارات و CI/CD

عند فشل الاختبارات، يقرأ Claude Code الأخطاء، يصلح الكود، ويعيد تشغيل مجموعة الاختبارات حتى تنجح جميعها. كما يراقب خطوط CI/CD على GitHub و GitLab ويرسل إصلاحات تلقائياً.

## تثبيت Claude Code

### متطلبات النظام

- **Node.js:** الإصدار 18 أو أحدث
- **Git:** الإصدار 2.38 أو أحدث

### طرق التثبيت

**الطريقة الأولى: عبر npm (الموصى بها)**


npm install -g @anthropic-ai/claude-code


> **ملاحظة:** لا تستخدم `sudo` مع npm، فهذا يسبب مشاكل في صلاحيات الملفات لاحقاً.

**الطريقة الثانية: عبر Homebrew (macOS)**


brew install --cask claude-code


**الطريقة الثالثة: عبر WinGet (Windows)**


winget install Anthropic.ClaudeCode


**الطريقة الرابعة: عبر المثبت الرسومي**

يتوفر مثبت رسومي لـ macOS و Windows على [claude.com/download](https://claude.com/download).

للدليل الشامل لجميع طرق التثبيت على مختلف أنظمة التشغيل، يمكنك مراجعة [تثبيت Claude Code: دليل التثبيت على Windows و macOS و Linux](/posts/claude-code-installation-guide).

**التحقق من التثبيت**


claude --version


### تسجيل الدخول إلى Claude Code

1. في الطرفية، شغل الأمر `claude`
2. عند المطالبة، اختر طريقة تسجيل الدخول المناسبة:
   - **لمشتركي Pro/Max:** سجل الدخول ببيانات Claude الخاصة بك
   - **لفرق Team/Enterprise:** استخدم `/login` ووافق على الطلب في المتصفح (يتضمن SSO)
   - **عبر API Key:** استخدم `export ANTHROPIC_API_KEY=sk-ant-...`
   - **عبر Amazon Bedrock:** استخدم `export CLAUDE_CODE_USE_BEDROCK=1`
   - **عبر Google Vertex AI:** استخدم `export CLAUDE_CODE_USE_VERTEX=1`

### أول مشروع مع Claude Code


cd path/to/your/project
claude


يقرأ Claude Code ملفاتك حسب الطلب، لذلك لا يوجد شيء لرفعه أو إرفاقه. ابدأه من داخل المستودع الذي تريد العمل عليه.



## التكامل مع VS Code و IDEs

يتوفر Claude Code كإضافة (Extension) رسمية لـ **VS Code**، بالإضافة إلى دعم **Cursor** و **JetBrains IDEs** (IntelliJ IDEA، PyCharm، WebStorm).

### إضافة VS Code لـ Claude Code

1. ابحث عن "Claude Code" في سوق إضافات VS Code
2. ثبّت الإضافة
3. أعد تشغيل VS Code بالكامل
4. افتح مجلد المشروع
5. افتح الشريط الجانبي لـ Claude وابدأ جلسة

**ميزات إضافة VS Code:**
- عرض التغييرات في الوقت الفعلي عبر لوحة جانبية مخصصة
- فروق مضمنة (Inline diffs) يمكنك قبولها أو رفضها
- نقاط استعادة للتراجع عن الأخطاء
- إشارات @-mentions للإشارة إلى ملفات محددة

> **ملاحظة مهمة:** استخدام الإضافة يستهلك من نفس حصة الاستخدام المشتركة عبر Claude و Claude Code.

للحصول على دليل كامل حول تثبيت الإضافة وتكوينها وأفضل الإعدادات، راجع [Claude Code مع VS Code: التكامل والإعدادات الكاملة](/posts/claude-code-vscode-integration).

## حالات استخدام Claude Code للمبرمجين

### 1. إصلاح اختبار فاشل

عند فشل اختبار، اكتب:

> "الاختبارات في tests/auth.test.ts فاشلة. هل يمكنك معرفة السبب وإصلاحها؟"

النتيجة: يقرأ Claude Code ملف الاختبار، ويتتبع مسار الكود، ويحدد المشكلة، ويقترح تعديلاً.

### 2. فهم كود غير مألوف

> "اشرح لي كيف تعمل منطق إعادة محاولة الدفع (payment retry logic)"

النتيجة: يحدد Claude Code الملفات ذات الصلة، ويقرأ التطبيق، ويشرح تدفق التحكم بلغة بسيطة.

### 3. البحث في قاعدة الكود

> "أين نتحقق من صحة عناوين البريد الإلكتروني في قاعدة الكود هذه؟"

النتيجة: يبحث Claude Code في المستودع، ويفتح الملفات المرشحة، ويعيد مسارات الملفات وأرقام الأسطر.

### 4. تحليل خطأ إنتاجي

> "أحصل على هذا الخطأ في الإنتاج: TypeError: Cannot read properties of undefined (reading 'sessionId') at validateSession (src/auth/session.ts:47). ما الذي يحدث؟"

النتيجة: يقرأ Claude Code الملفات المشار إليها في التتبع، ويشرح سبب المشكلة، ويقترح إصلاحاً.

### 5. ترحيل كود كبير

مثل ترجمة Bun من Zig إلى Rust - 750,000 سطر في 11 يوماً.

## مقارنة Claude Code مع أدوات أخرى

**Claude Code:** وكيل طرفية (CLI Agent) - سياق قاعدة الكود بالكامل - وكالة عالية - سعر $17-200/شهر - الأفضل للمشاريع الكبيرة والترحيلات

**Cursor:** IDE-native - سياق قاعدة الكود بالكامل - وكالة متوسطة - سعر $20-40/شهر - الأفضل لتجربة محرر متكاملة

**GitHub Copilot:** مكمل كود - سياق الملف المفتوح - وكالة منخفضة - سعر $10-39/شهر - الأفضل للإكمال السريع اليومي

**الخلاصة:** Claude Code هو أقوى وكيل برمجة مستقل للعمل الهندسي اليومي، Cursor هو أفضل تجربة IDE-native، وGitHub Copilot لا يزال الخيار الأكثر أماناً للمؤسسات.

## نصائح للمبتدئين في Claude Code

1. **ابدأ بمهمة صغيرة**: جرب إضافة توثيق (Docstring) لدالة بسيطة لتتعرف على سير العمل.
2. **استخدم أوامر البداية السريعة**: جرب الأوامر الخمسة في دليل اليوم الأول.
3. **افهم قاعدة الأمان**: سيسأل Claude Code دائماً قبل تغيير الملفات أو تشغيل الأوامر.
4. **استخدم CLAUDE.md**: أنشئ ملف `CLAUDE.md` في جذر مشروعك لتوفير سياق دائم.
5. **جرب `/init`**: استخدم هذا الأمر لإنشاء نقطة بداية تلقائية لمشروعك.
