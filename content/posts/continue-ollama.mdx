# Continue مع Ollama: التشغيل المحلي للذكاء الاصطناعي 2026

في عام 2026، أصبحت أدوات البرمجة بالذكاء الاصطناعي جزءاً أساسياً من سير عمل المطورين. لكن العديد من هذه الأدوات تتطلب اتصالاً بالإنترنت ودفع تكاليف API. هنا يأتي دور **Continue مع Ollama**، وهو تكامل يسمح لك بتشغيل مساعد برمجة بالذكاء الاصطناعي **محلياً** و**مجاناً** و**بدون إنترنت**.

هذا الدليل يقدم لك كل ما تحتاج معرفته عن تشغيل Continue.dev مع Ollama: التثبيت، تحميل النماذج، التهيئة، والاستخدام الفعال.

---

## 1. ما هو Continue مع Ollama؟

**Continue مع Ollama** هو تكامل بين إضافة **Continue.dev** في VS Code و **Ollama**، وهي أداة لتشغيل نماذج الذكاء الاصطناعي محلياً.

### 1.1 ما هو Ollama؟

**Ollama** هو تطبيق مفتوح المصدر يسمح لك بتشغيل نماذج لغة كبيرة (LLMs) محلياً على جهازك. يدعم Ollama العديد من النماذج المفتوحة المصدر مثل:
- **CodeLlama**: نماذج برمجة من Meta
- **DeepSeek-Coder**: نماذج برمجة متقدمة
- **Mistral**: نماذج عامة سريعة
- **Phi-3**: نماذج خفيفة من Microsoft
- **Llama 3**: أحدث نماذج Meta

### 1.2 لماذا تستخدم Continue مع Ollama؟

| الميزة | الفائدة |
|--------|---------|
| **مجاني تماماً** | لا حاجة لدفع تكاليف API |
| **بدون إنترنت** | يعمل في أي مكان |
| **خصوصية كاملة** | بياناتك لا تخرج من جهازك |
| **سرعة عالية** | استجابة سريعة بدون تأخير شبكة |
| **مرونة** | اختر النموذج المناسب لمهامك |
| **تحكم كامل** | تخصيص الإعدادات حسب احتياجاتك |

|---------|----------|

| حجم النموذج | RAM | VRAM (GPU) | المساحة |
|-------------|-----|------------|---------|

| **13B (متوسط)** | 16GB | 8GB | 8GB |
| **34B (كبير)** | 32GB | 16GB | 20GB |

## 3. تثبيت Ollama

### 3.1 تثبيت Ollama على Windows

1. قم بزيارة [الموقع الرسمي لـ Ollama](https://ollama.ai).
2. حمّل ملف التثبيت لـ Windows.
3. شغّل الملف واتبع التعليمات.
4. بعد التثبيت، افتح **Command Prompt** أو **PowerShell**.
5. تحقق من التثبيت: `ollama --version`

### 3.2 تثبيت Ollama على macOS

```bash
curl -fsSL https://ollama.ai/install.sh | sh
```

### 3.3 تثبيت Ollama على Linux

```bash
curl -fsSL https://ollama.ai/install.sh | sh
```

### 3.4 تشغيل Ollama

بعد التثبيت، شغّل Ollama في الخلفية:
```bash
ollama serve
```

> **للمزيد**: راجع [تثبيت Ollama: دليل كامل](/posts/ollama-installation-guide).

## 4. تحميل النماذج في Ollama

### 4.1 قائمة النماذج المتوفرة

| النموذج | الحجم | الوصف |
|---------|-------|-------|
| `codellama:7b` | 7B | نموذج برمجة عام من Meta |
| `codellama:13b` | 13B | نموذج برمجة متوسط |
| `codellama:34b` | 34B | نموذج برمجة كبير (يتطلب موارد عالية) |
| `deepseek-coder:6.7b` | 6.7B | نموذج برمجة متقدم من DeepSeek |
| `deepseek-coder:33b` | 33B | نموذج برمجة كبير متقدم |
| `mistral:7b` | 7B | نموذج عام سريع |
| `phi3:mini` | 3.8B | نموذج خفيف من Microsoft |
| `llama3:8b` | 8B | أحدث نموذج من Meta |

### 4.2 تحميل نموذج

**الأمر العام**:
```bash
ollama pull [اسم_النموذج]
```

**أمثلة**:
```bash
# تحميل CodeLlama 7B
ollama pull codellama:7b

# تحميل DeepSeek-Coder 6.7B
ollama pull deepseek-coder:6.7b

# تحميل Mistral 7B
ollama pull mistral:7b
```

### 4.3 عرض النماذج المحملة

```bash
ollama list
```

### 4.4 حذف نموذج

```bash
ollama rm [اسم_النموذج]
```

## 5. تهيئة Continue.dev مع Ollama

### 5.1 تثبيت Continue.dev

إذا لم تقم بتثبيت Continue.dev بعد:

1. افتح VS Code.
2. اذهب إلى Extensions (`Ctrl+Shift+X`).
3. ابحث عن **"Continue"** واضغط تثبيت.

> **للمزيد**: راجع [تثبيت Continue.dev في VS Code](/posts/continue-installation).

### 5.2 إضافة نموذج Ollama إلى Continue

**الطريقة الأولى: من خلال واجهة Continue**

1. افتح Continue (أيقونة في الشريط الجانبي).
2. اضغط على أيقونة **الإعدادات** (الترس).
3. اختر **"Add Model"**.
4. اختر **"Ollama"**.
5. اختر النموذج المحمّل (مثل `codellama:7b`).

**الطريقة الثانية: من خلال ملف config.json**

1. في واجهة Continue، اضغط على **"Open Config"**.
2. أضف النموذج في قسم `models`:

```json
{
  "models": [
    {
      "title": "CodeLlama 7B",
      "provider": "ollama",
      "model": "codellama:7b"
    }
  ]
}
```

### 5.3 إعداد نموذج الإكمال التلقائي

لتحسين الأداء، يمكنك تعيين نموذج أخف للإكمال التلقائي:

```json
{
  "tabAutocompleteModel": {
    "title": "CodeLlama 7B",
    "provider": "ollama",
    "model": "codellama:7b"
  }
}
```

### 5.4 إعداد النماذج المتعددة

يمكنك إضافة نماذج متعددة واستخدامها حسب المهمة:

```json
{
  "models": [
    {
      "title": "CodeLlama 7B",
      "provider": "ollama",
      "model": "codellama:7b"
    },
    {
      "title": "DeepSeek-Coder 6.7B",
      "provider": "ollama",
      "model": "deepseek-coder:6.7b"
    },
    {
      "title": "Mistral 7B",
      "provider": "ollama",
      "model": "mistral:7b"
    }
  ]
}
```

## 6. استخدام Continue مع Ollama

### 6.1 المحادثة (Chat)

**فتح نافذة المحادثة**:
- `Ctrl+Shift+L` (Windows/Linux) أو `Cmd+Shift+L` (macOS)

**الأمر**:
> "اكتب دالة Python لحساب الأعداد الأولية حتى 1000."

**النتيجة**: كود جاهز مع شرح.

### 6.2 الإكمال التلقائي (Autocomplete)

أثناء كتابة الكود، ستظهر اقتراحات تلقائية من النموذج المحلي.

**مفتاح الاختصار**: `Tab` لقبول الاقتراح.

### 6.3 تصحيح الأخطاء (Debugging)

**الأمر**:
> "هذا الكود ينتج خطأ. ساعدني في إصلاحه: [الكود]"

**النتيجة**: تحليل الخطأ مع حل مقترح.

### 6.4 إعادة الهيكلة (Refactoring)

**الأمر**:
> "أعد هيكلة هذه الدالة لتكون أكثر كفاءة: [الكود]"

**النتيجة**: كود محسّن مع شرح التغييرات.

### 6.5 توليد الاختبارات

**الأمر**:
> "اكتب اختبارات وحدة لهذه الدالة باستخدام pytest: [الكود]"

**النتيجة**: اختبارات جاهزة للتشغيل.

## 8. تحسين أداء Continue مع Ollama

### 8.1 استخدام تسريع GPU

إذا كان لديك بطاقة رسوميات NVIDIA:

1. تأكد من تثبيت **CUDA**.
2. شغّل Ollama مع دعم GPU (افتراضي).
3. تحقق من استخدام GPU: `ollama ps`

### 8.2 تقليل حجم السياق

في إعدادات Continue، قلل `contextLength` لتسريع الأداء:

```json
{
  "contextLength": 2048
}
```

### 8.3 استخدام نموذجين مختلفين

- **نموذج خفيف**: للإكمال التلقائي (مثل `codellama:7b`).
- **نموذج ثقيل**: للمحادثات والمهام المعقدة (مثل `deepseek-coder:6.7b`).

### 8.4 إعداد ذاكرة التخزين المؤقت

في Ollama، يمكنك زيادة حجم الذاكرة المؤقتة:

```bash
export OLLAMA_NUM_PARALLEL=4
export OLLAMA_MAX_LOADED_MODELS=2
```

### 8.5 تحديث Ollama بانتظام

```bash
# تحديث Ollama
ollama update
```

## 10. حل المشكلات الشائعة

### 10.1 Ollama لا يعمل

**الحل**:
1. تأكد من تشغيل Ollama: `ollama serve`
2. تحقق من التثبيت: `ollama --version`
3. أعد تشغيل الجهاز.

### 10.2 النموذج لا يستجيب

**الحل**:
1. تحقق من تحميل النموذج: `ollama list`
2. حمّل النموذج: `ollama pull codellama:7b`
3. تأكد من صحة اسم النموذج في إعدادات Continue.

### 10.3 الأداء بطيء

**الحل**:
1. استخدم نموذجاً أصغر (7B بدلاً من 13B).
2. فعّل تسريع GPU.
3. قلل حجم السياق.
4. أغلق التطبيقات الأخرى التي تستهلك الذاكرة.

### 10.4 خطأ في الذاكرة

**الحل**:
1. استخدم نموذجاً أصغر.
2. أضف ذاكرة افتراضية (Swap) على النظام.
3. أغلق التطبيقات غير الضرورية.

### 10.5 Continue لا يرى النموذج

**الحل**:
1. تأكد من أن Ollama يعمل.
2. تحقق من اسم النموذج في إعدادات Continue.
3. أعد تشغيل Continue أو VS Code.

> **للمزيد**: راجع [حل مشكلات ChatGPT الشائعة](/posts/chatgpt-common-problems).

## 11. الخلاصة

استخدام Continue مع Ollama يمنحك قوة الذكاء الاصطناعي في البرمجة دون الحاجة إلى إنترنت أو تكاليف. سواء كنت مطوراً مستقلاً، أو تعمل في بيئة مغلقة، أو تريد خصوصية كاملة، فإن هذا التكامل هو الحل المثالي.

**الخلاصة في نقاط**:
- **مجاني تماماً**: بدون تكاليف API.
- **بدون إنترنت**: يعمل في أي مكان.
- **خصوصية كاملة**: بياناتك لا تخرج من جهازك.
- **نماذج متعددة**: اختر النموذج المناسب لجهازك ومهامك.
- **سهولة التهيئة**: بضع خطوات فقط.
- **أداء ممتاز**: مع إعدادات محسّنة.

**توصيتنا**: ابدأ بـ CodeLlama 7B إذا كان جهازك محدوداً، أو DeepSeek-Coder 6.7B إذا كنت تبحث عن أداء برمجي ممتاز. جرّب نماذج مختلفة واختر ما يناسب احتياجاتك.
