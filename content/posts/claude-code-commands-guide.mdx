# أوامر Claude Code الأساسية: دليل سريع للمبرمجين 2026

Claude Code هو أداة برمجة وكيلة (Agentic Coding Tool) تعمل عبر سطر الأوامر. لفهم قوتها الكاملة، تحتاج إلى معرفة الأوامر الأساسية والمتقدمة.

هذا الدليل يغطي جميع أوامر Claude Code التي ستحتاجها في عملك اليومي، مع أمثلة عملية لكل أمر.

للتعرف على Claude Code بشكل عام، يمكنك قراءة المقالة الرئيسية [Claude Code: الدليل الشامل للمبتدئين والمحترفين في 2026](/posts/claude-code-guide-2026).

## الأوامر الأساسية (CLI)

### claude

الأمر الرئيسي لبدء جلسة Claude Code.

**الاستخدام:**

```bash
claude
```

**الوصف:** يبدأ جلسة تفاعلية مع Claude Code في الطرفية.

**مثال:**

```bash
cd my-project
claude
```

### claude --version

عرض إصدار Claude Code المثبت.

**الاستخدام:**

```bash
claude --version
```

**الوصف:** يعرض رقم الإصدار الحالي لـ Claude Code.

**مثال:**

```bash
claude --version
# الناتج: 1.0.0
```

### claude --help

عرض قائمة الأوامر المتاحة والمساعدة.

**الاستخدام:**

```bash
claude --help
```

**الوصف:** يعرض جميع الأوامر المتاحة مع وصف مختصر لكل أمر.

**مثال:**

```bash
claude --help
```

### claude login

تسجيل الدخول إلى حساب Claude.

**الاستخدام:**

```bash
claude login
```

**الوصف:** يفتح المتصفح لتسجيل الدخول بحساب Claude الخاص بك.

**مثال:**

```bash
claude login
# يفتح المتصفح لتسجيل الدخول
```

### claude logout

تسجيل الخروج من حساب Claude.

**الاستخدام:**

```bash
claude logout
```

**الوصف:** يسجل الخروج من حساب Claude الحالي.

**مثال:**

```bash
claude logout
```

### claude config

عرض أو تعديل إعدادات Claude Code.

**الاستخدام:**

```bash
claude config
claude config set [key] [value]
claude config get [key]
```

**الوصف:** يعرض الإعدادات الحالية أو يعدلها.

**مثال:**

```bash
claude config
claude config set model sonnet
claude config get model
```

## أوامر الجلسة (داخل Claude Code)

### /help

عرض قائمة الأوامر المتاحة داخل الجلسة.

**الاستخدام:**

```
/help
```

**الوصف:** يعرض جميع الأوامر المتاحة داخل جلسة Claude Code.

**مثال:**

```
/help
# يعرض قائمة بالأوامر المتاحة
```

### /exit

إنهاء جلسة Claude Code الحالية.

**الاستخدام:**

```
/exit
```

**الوصف:** ينهي الجلسة الحالية ويعود إلى الطرفية.

**مثال:**

```
/exit
# إنهاء الجلسة
```

### /clear

مسح محادثة الجلسة الحالية.

**الاستخدام:**

```
/clear
```

**الوصف:** يمسح تاريخ المحادثة ويعيد ضبط السياق.

**مثال:**

```
/clear
# مسح المحادثة
```

### /resume

استئناف جلسة سابقة.

**الاستخدام:**

```
/resume [session-id]
```

**الوصف:** يستأنف جلسة سابقة باستخدام معرف الجلسة.

**مثال:**

```
/resume
# يعرض قائمة بالجلسات السابقة
/resume session_123
# يستأنف الجلسة رقم 123
```

### /new

بدء جلسة جديدة.

**الاستخدام:**

```
/new
```

**الوصف:** يبدأ جلسة جديدة مع إعادة ضبط السياق.

**مثال:**

```
/new
# بدء جلسة جديدة
```

## أوامر الملفات والمشروع

### /file

قراءة ملف محدد.

**الاستخدام:**

```
/file [path]
```

**الوصف:** يقرأ محتوى ملف محدد ويضيفه إلى السياق.

**مثال:**

```
/file src/utils/helpers.js
# يقرأ ملف helpers.js
```

### /write

كتابة أو تعديل ملف.

**الاستخدام:**

```
/write [path] [content]
```

**الوصف:** يكتب محتوى إلى ملف محدد أو يعدله.

**مثال:**

```
/write src/utils/math.js "export const add = (a, b) => a + b;"
# يكتب دالة add في ملف math.js
```

### /read

قراءة ملفات متعددة.

**الاستخدام:**

```
/read [path1] [path2] ...
```

**الوصف:** يقرأ عدة ملفات ويضيفها إلى السياق.

**مثال:**

```
/read src/utils/*.js
# يقرأ جميع ملفات JS في مجلد utils
```

### @-mentions

الإشارة إلى ملفات محددة في السؤال.

**الاستخدام:**

```
@[path]
```

**الوصف:** يضيف ملف محدد إلى سياق المحادثة.

**مثال:**

> "عدّل @src/auth/login.js لتحسين الأمان"

> "اشرح لي @src/utils/helpers.js"

## الأوامر المتقدمة

### /init

تهيئة مشروع مع Claude Code.

**الاستخدام:**

```
/init
```

**الوصف:** ينشئ ملف CLAUDE.md في جذر المشروع بإعدادات مبدئية.

**مثال:**

```
/init
# إنشاء CLAUDE.md
```

### /agents

عرض الوكلاء النشطين.

**الاستخدام:**

```
/agents
```

**الوصف:** يعرض قائمة بالوكلاء الفرعيين النشطين في الجلسة الحالية.

**مثال:**

```
/agents
# يعرض قائمة الوكلاء
```

### /auto

تفعيل أو تعطيل الوضع التلقائي.

**الاستخدام:**

```
/auto [on|off]
```

**الوصف:** يفعل أو يعطل الوضع التلقائي الذي يوافق تلقائياً على التغييرات.

**مثال:**

```
/auto on
# تفعيل الوضع التلقائي
```

### /checkpoint

إنشاء نقطة استعادة يدوية.

**الاستخدام:**

```
/checkpoint [name]
```

**الوصف:** ينشئ نقطة استعادة يمكن الرجوع إليها لاحقاً.

**مثال:**

```
/checkpoint "Before major refactor"
# إنشاء نقطة استعادة
```

### /undo

التراجع عن آخر تغيير.

**الاستخدام:**

```
/undo
```

**الوصف:** يتراجع عن آخر تغيير تم تطبيقه.

**مثال:**

```
/undo
# التراجع عن آخر تغيير
```

### /redo

إعادة تطبيق آخر تغيير تم التراجع عنه.

**الاستخدام:**

```
/redo
```

**الوصف:** يعيد تطبيق آخر تغيير تم التراجع عنه.

**مثال:**

```
/redo
# إعادة تطبيق التغيير
```

## الأوامر البيئية

### claude --config

تحديد ملف تكوين مخصص.

**الاستخدام:**

```bash
claude --config [path]
```

**الوصف:** يستخدم ملف تكوين مخصص بدلاً من الإعدادات الافتراضية.

**مثال:**

```bash
claude --config ~/my-config.json
```

### claude --model

اختيار نموذج محدد.

**الاستخدام:**

```bash
claude --model [model-name]
```

**الوصف:** يحدد النموذج المستخدم (Sonnet، Opus، إلخ).

**مثال:**

```bash
claude --model opus
```

### claude --no-auto

تعطيل الوضع التلقائي في جلسة CLI.

**الاستخدام:**

```bash
claude --no-auto
```

**الوصف:** يبدأ جلسة بدون الوضع التلقائي، مما يتطلب موافقة يدوية على كل تغيير.

**مثال:**

```bash
claude --no-auto
```

### claude --verbose

عرض معلومات تفصيلية أثناء التشغيل.

**الاستخدام:**

```bash
claude --verbose
```

**الوصف:** يعرض معلومات إضافية عن ما يفعله Claude Code.

**مثال:**

```bash
claude --verbose
```

## أمثلة عملية للأوامر

### مثال 1: بدء جلسة مع مشروع

```bash
cd my-project
claude
```

ثم داخل الجلسة:

> "اكتب دالة في src/utils/math.js لحساب المتوسط الحسابي"

### مثال 2: قراءة ملف وطلب تعديل

```bash
claude
/file src/app.js
```

> "حسّن أداء هذه الدالة باستخدام memoization"

### مثال 3: العمل على ملفات متعددة

```bash
claude
/read src/components/*.jsx
```

> "أعد هيكلة هذه المكونات لاستخدام React Hooks"

### مثال 4: إنشاء نقطة استعادة قبل تغيير كبير

```bash
claude
/checkpoint "Before migration"
```

> "رحّل قاعدة الكود من JavaScript إلى TypeScript"

### مثال 5: التراجع عن تغيير خاطئ

```bash
claude
/checkpoint "After refactor"
```

> "لدي خطأ في الكود الجديد. أصلحه."

إذا لم ينجح الإصلاح:

```bash
/undo
```

## متغيرات البيئة

### ANTHROPIC_API_KEY

استخدام مفتاح API بدلاً من حساب Claude.

```bash
export ANTHROPIC_API_KEY=sk-ant-...
claude
```

### CLAUDE_CODE_USE_BEDROCK

استخدام Amazon Bedrock للمصادقة.

```bash
export CLAUDE_CODE_USE_BEDROCK=1
claude
```

### CLAUDE_CODE_USE_VERTEX

استخدام Google Vertex AI للمصادقة.

```bash
export CLAUDE_CODE_USE_VERTEX=1
claude
```

### CLAUDE_CODE_CONFIG

تحديد موقع ملف التكوين.

```bash
export CLAUDE_CODE_CONFIG=~/my-config.json
claude
```

## نصائح لاستخدام الأوامر بفعالية

### 1. استخدم @-mentions لتحديد الملفات

بدلاً من كتابة مسار الملف في سؤالك، استخدم @ للإشارة إليه:

> "أصلح الخطأ في @src/app.js"

### 2. استخدم /read لقراءة ملفات متعددة

إذا كنت تعمل على عدة ملفات، استخدم /read لقراءتها جميعاً:

```
/read src/utils/*.js src/constants/*.js
```

### 3. أنشئ نقاط استعادة قبل التغييرات الكبيرة

استخدم /checkpoint قبل التغييرات المهمة لتتمكن من التراجع بسهولة.

### 4. استخدم /init للمشاريع الجديدة

عند بدء مشروع جديد مع Claude Code، استخدم /init لإنشاء ملف CLAUDE.md تلقائياً.

### 5. تعلم الأوامر المختصرة

- `Ctrl+C`: إنهاء الجلسة
- `Ctrl+L`: مسح الطرفية
- `Tab`: إكمال تلقائي للمسارات والملفات

## حل مشاكل الأوامر الشائعة

### مشكلة: "command not found: claude"

**السبب:** لم يتم تثبيت Claude Code بشكل صحيح.

**الحل:** أعد تثبيت Claude Code باستخدام npm:

```bash
npm install -g @anthropic-ai/claude-code
```

### مشكلة: "/command not recognized"

**السبب:** الأمر غير موجود أو أخطأت في كتابته.

**الحل:** استخدم `/help` لعرض قائمة الأوامر المتاحة.

### مشكلة: "Authentication failed"

**السبب:** انتهت صلاحية جلسة تسجيل الدخول.

**الحل:** أعد تسجيل الدخول:

```bash
claude login
```

### مشكلة: "File not found"

**السبب:** المسار غير صحيح أو الملف غير موجود.

**الحل:** تحقق من المسار باستخدام `ls` أو `dir`.

## الخلاصة

أوامر Claude Code بسيطة وقوية. أتقن الأوامر الأساسية أولاً، ثم انتقل إلى الأوامر المتقدمة حسب احتياجاتك.

**الخلاصة في 5 نقاط:**

- **الأساسيات:** `claude`، `claude --version`، `claude --help`
- **تسجيل الدخول:** `claude login` و `claude logout`
- **داخل الجلسة:** `/help`، `/exit`، `/clear`، `/resume`
- **الملفات:** `/file`، `/write`، `/read`، `@-mentions`
- **المتقدمة:** `/init`، `/agents`، `/auto`، `/checkpoint`، `/undo`

جرب هذه الأوامر في مشروعك اليوم وسترى كيف تزيد إنتاجيتك.
