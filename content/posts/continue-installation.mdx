# تثبيت Continue.dev في VS Code: الدليل الكامل 2026

**Continue.dev** هو مساعد برمجة بالذكاء الاصطناعي مفتوح المصدر يدمج بسلاسة في VS Code. سواء كنت تريد استخدام نماذج سحابية مثل GPT-4 أو نماذج محلية عبر Ollama، فإن تثبيت Continue.dev هو الخطوة الأولى لتحويل تجربة البرمجة الخاصة بك.

هذا الدليل يقدم لك كل ما تحتاج معرفته عن تثبيت Continue.dev في VS Code: خطوات التثبيت، التهيئة، إعداد النماذج، وحل المشكلات الشائعة.

---

## 1. متطلبات التثبيت

### 1.1 متطلبات النظام

| المتطلب | التفاصيل |
|---------|----------|
| **VS Code** | الإصدار 1.70 أو أحدث |
| **نظام التشغيل** | Windows 10/11، macOS 10.15+، Linux |
| **الاتصال بالإنترنت** | للتحميل الأولي (يمكن استخدامه بدون إنترنت لاحقاً مع النماذج المحلية) |
| **الذاكرة** | 4GB RAM كحد أدنى (8GB موصى بها للنماذج المحلية) |
| **المساحة** | 50 ميجابايت للإضافة + مساحة للنماذج المحلية (حسب النموذج) |

### 1.2 متطلبات إضافية للنماذج المحلية

إذا كنت تريد استخدام نماذج محلية عبر **Ollama**:
- ثبّت [Ollama](https://ollama.ai) على جهازك.
- حمّل نموذجاً مثل `codellama` أو `deepseek-coder`.

> **للمزيد**: راجع [تثبيت Ollama: دليل كامل](/posts/ollama-installation-guide).

---

## 2. تثبيت Continue.dev من متجر إضافات VS Code

### 2.1 الطريقة الأولى: من خلال واجهة VS Code

**خطوات التثبيت**:

1. **افتح VS Code**.
2. **افتح علامة التبويب Extensions**:
   - اضغط `Ctrl+Shift+X` (Windows/Linux) أو `Cmd+Shift+X` (macOS).
   - أو اضغط على أيقونة المربعات في الشريط الجانبي الأيسر.
3. **ابحث عن Continue**:
   - اكتب `Continue` في شريط البحث.
   - ابحث عن الإضافة التي تحمل اسم **"Continue"** من مطور **"Continue Dev"**.
4. **اضغط على تثبيت (Install)**.
5. **انتظر حتى يكتمل التحميل** (يستغرق بضع ثوانٍ).
6. **بعد التثبيت**، ستظهر أيقونة Continue في الشريط الجانبي الأيسر.

### 2.2 الطريقة الثانية: من خلال سطر الأوامر

إذا كنت تفضل التثبيت عبر سطر الأوامر:

```bash
code --install-extension continue.continue
```

### 2.3 الطريقة الثالثة: من خلال متجر الإضافات عبر المتصفح

1. اذهب إلى [متجر إضافات VS Code](https://marketplace.visualstudio.com/items?itemName=Continue.continue).
2. اضغط على **"Install"**.
3. اتبع التعليمات لفتح الرابط في VS Code.

---

## 3. التحقق من التثبيت

بعد التثبيت، تأكد من أن الإضافة تعمل بشكل صحيح:

1. **ابحث عن أيقونة Continue** في الشريط الجانبي الأيسر.
2. **اضغط على الأيقونة** لفتح واجهة Continue.
3. **إذا ظهرت واجهة الترحيب**، فهذا يعني أن التثبيت تم بنجاح.

## 4. التهيئة الأولية لـ Continue.dev

### 4.1 اختيار النماذج

عند فتح Continue لأول مرة، سيطلب منك تهيئة النماذج. لديك ثلاثة خيارات رئيسية:

#### الخيار 1: استخدام نماذج سحابية (OpenAI، Claude، Gemini)

**الخطوات**:
1. احصل على مفتاح API من مزود الخدمة (مثل OpenAI).
2. في واجهة Continue، اختر **"Add Model"**.
3. اختر المزود (مثل OpenAI).
4. أدخل مفتاح API.
5. اختر النموذج (مثل `gpt-4`).

#### الخيار 2: استخدام نماذج محلية (Ollama) - موصى به للمجاني والخاص

**الخطوات**:
1. تأكد من تثبيت [Ollama](https://ollama.ai).
2. حمّل نموذجاً: `ollama pull codellama:7b`.
3. في Continue، اختر **"Add Model"** > **"Ollama"**.
4. اختر النموذج الذي حمّلته.

> **للمزيد**: راجع [Continue مع Ollama: التشغيل المحلي](/posts/continue-ollama).

#### الخيار 3: نماذج مخصصة

يمكنك إضافة أي نموذج عبر API مخصص:

```json
{
  "title": "Custom Model",
  "provider": "custom",
  "model": "my-model",
  "apiBase": "http://localhost:5000/v1"
}
```

### 4.2 إعداد نموذج الإكمال التلقائي

يمكنك تعيين نموذج مختلف للإكمال التلقائي (عادةً نموذج أخف وأسرع):

1. في إعدادات Continue، ابحث عن `tabAutocompleteModel`.
2. اختر نموذجاً خفيفاً مثل `codellama:7b` أو `gpt-3.5-turbo`.

### 4.3 إعداد السياق

يمكنك تحديد المصادر التي يستخدمها Continue لفهم السياق:

- **Codebase**: قراءة ملفات المشروع.
- **Terminal**: فهم الأخطاء من المحطة الطرفية.
- **Docs**: قراءة ملفات README والوثائق.

## 5. ملف الإعدادات (config.json)

يمكنك تخصيص Continue.dev عبر ملف `config.json`. لفتح الملف:

1. في واجهة Continue، اضغط على أيقونة **الإعدادات** (الترس).
2. اختر **"Open Config"**.

**مثال لملف إعدادات متكامل**:

```json
{
  "models": [
    {
      "title": "GPT-4",
      "provider": "openai",
      "model": "gpt-4",
      "apiKey": "YOUR_API_KEY"
    },
    {
      "title": "CodeLlama",
      "provider": "ollama",
      "model": "codellama:7b"
    }
  ],
  "tabAutocompleteModel": {
    "title": "CodeLlama",
    "provider": "ollama",
    "model": "codellama:7b"
  },
  "contextProviders": [
    {
      "name": "codebase",
      "params": {}
    },
    {
      "name": "terminal",
      "params": {}
    }
  ],
  "slashCommands": [
    {
      "name": "explain",
      "description": "شرح الكود المحدد",
      "prompt": "اشرح الكود التالي بالتفصيل:"
    },
    {
      "name": "test",
      "description": "توليد اختبارات وحدة",
      "prompt": "اكتب اختبارات وحدة للكود التالي:"
    }
  ]
}
```

## 6. اختصارات لوحة المفاتيح الأساسية

| الاختصار (Windows/Linux) | الاختصار (macOS) | الوظيفة |
|--------------------------|------------------|---------|
| `Ctrl+Shift+L` | `Cmd+Shift+L` | فتح نافذة المحادثة |
| `Ctrl+Shift+I` | `Cmd+Shift+I` | فتح محادثة مضمّنة (Inline Chat) |
| `Tab` | `Tab` | قبول الاقتراح التلقائي |
| `Ctrl+Space` | `Cmd+Space` | عرض اقتراحات إضافية |
| `Ctrl+Shift+Enter` | `Cmd+Shift+Enter` | تطبيق الاقتراح في الملف |

## 7. التحقق من التثبيت والتشغيل

### 7.1 اختبار الإكمال التلقائي

1. افتح ملف كود (مثل `index.js` أو `main.py`).
2. ابدأ كتابة دالة بسيطة، مثلاً:
   ```python
   def calculate_average(
   ```
3. إذا ظهر اقتراح من Continue، فهذا يعني أن الإكمال التلقائي يعمل.

### 7.2 اختبار المحادثة

1. اضغط `Ctrl+Shift+L` لفتح نافذة المحادثة.
2. اكتب سؤالاً بسيطاً:
   > "اكتب دالة Python لجمع الأعداد الزوجية من 1 إلى 100."
3. إذا حصلت على رد، فهذا يعني أن المحادثة تعمل.

## 8. حل المشكلات الشائعة

### 8.1 Continue.dev لا يظهر في الشريط الجانبي

**الحل**:
1. أعد تحميل VS Code.
2. تأكد من تثبيت الإضافة بشكل صحيح.
3. جرب تعطيل الإضافة وإعادة تمكينها.
4. إذا استمرت المشكلة، احذف الإضافة وأعد تثبيتها.

### 8.2 النموذج لا يستجيب

**الحل**:
1. تحقق من مفتاح API (للنماذج السحابية).
2. تأكد من تشغيل Ollama (للنماذج المحلية).
3. تحقق من صحة اسم النموذج في الإعدادات.
4. أعد تشغيل VS Code.

### 8.3 الإكمال التلقائي بطيء

**الحل**:
1. استخدم نموذجاً أخف (مثل `codellama:7b` بدلاً من `codellama:34b`).
2. قلل حجم السياق في الإعدادات.
3. إذا كنت تستخدم نموذجاً سحابياً، تأكد من اتصال إنترنت جيد.

### 8.4 خطأ في تحميل النموذج المحلي

**الحل**:
1. تأكد من تثبيت Ollama وتشغيله.
2. تحقق من تحميل النموذج: `ollama list`.
3. إذا لم يكن النموذج موجوداً، حمّله: `ollama pull codellama:7b`.

### 8.5 Continue.dev يتعارض مع إضافات أخرى

**الحل**:
1. عطّل الإضافات الأخرى مؤقتاً لتحديد المتعارض.
2. أضف Continue.dev إلى قائمة الإضافات الموثوقة.
3. تحديث VS Code إلى أحدث إصدار.

> **للمزيد**: راجع [أفضل إعدادات Continue للمبرمجين](/posts/continue-best-settings).

## 9. نصائح لتحسين أداء Continue.dev

### 9.1 استخدم النماذج المحلية للسرعة والخصوصية

النماذج المحلية عبر Ollama أسرع وأكثر خصوصية من النماذج السحابية.

### 9.2 خصص السياق

حدد الملفات والمجلدات التي يقرأها Continue لتقليل الحمل وتحسين الدقة.

### 9.3 استخدم نموذجين مختلفين

- **نموذج خفيف**: للإكمال التلقائي (مثل `codellama:7b`).
- **نموذج ثقيل**: للمحادثات والمهام المعقدة (مثل `gpt-4`).

### 9.4 حدّث الإضافة بانتظام

تأكد من تحديث Continue.dev إلى أحدث إصدار للحصول على أحدث الميزات والتحسينات.

## 10. الخلاصة

تثبيت Continue.dev في VS Code هو عملية بسيطة وسريعة تفتح لك عالمًا من الإمكانيات البرمجية المدعومة بالذكاء الاصطناعي. سواء اخترت النماذج السحابية أو المحلية، يوفر Continue.dev تجربة برمجة متطورة ومجانية بالكامل.

**الخلاصة في نقاط**:
- **التثبيت**: من متجر إضافات VS Code في ثوانٍ.
- **التهيئة**: إضافة نماذج (سحابية أو محلية) عبر واجهة بسيطة.
- **النماذج المحلية**: عبر Ollama للاستخدام المجاني والخاص.
- **الميزات**: إكمال تلقائي، محادثة، تصحيح أخطاء، إعادة هيكلة.
- **التخصيص**: عبر ملف config.json.
- **حل المشكلات**: معظم المشكلات تحل بإعادة تحميل VS Code أو التحقق من النماذج.

**توصيتنا**: ابدأ باستخدام النماذج المحلية عبر Ollama لتجربة مجانية وآمنة. ثم جرّب النماذج السحابية للمهام التي تتطلب فهماً أعمق.
